# Workflow: Research & Report

## Trigger
User provides a topic they want researched.

## Step 1 — Intake (always first, no exceptions)

Before any research, ask the user these questions in a single message:

1. **Scope** — How broad or narrow? (e.g., "overview of the field" vs "how X compares to Y")
2. **Audience** — Who will read this? (expert, general public, yourself, clients)
3. **Depth** — Surface-level summary, mid-depth analysis, or deep dive with sources?
4. **Angle** — Any specific aspect to emphasize? (pros/cons, how-to, trends, history, comparison)
5. **Output format** — Full report, executive summary, bullet-point brief, or slide-ready outline?

Wait for the user's answers before proceeding.

## Step 2 — Research Plan

Show the user a short plan before executing:
- List 4–6 specific search queries you will use
- Name the source types you'll prioritize (news, academic, official docs, expert blogs)
- State the report structure you'll follow (reference: `ressources/report-template.md`)

Get a quick confirmation ("looks good" or adjustments) before searching.

## Step 3 — Research Execution

Follow the methodology in `ressources/research-methodology.md`.

Key rules:
- Run at minimum 5 distinct searches (vary query phrasing, not just keywords)
- Fetch and read at least 3 full source pages — don't rely on snippets alone
- Track every source (title, URL, date) as you go
- Flag conflicting information — don't silently pick one side
- Separate **facts** (verifiable) from **opinions/analysis** (attributed)

## Step 4 — Synthesis

Before writing the report:
- Group findings into 3–5 themes
- Identify the single most important insight
- Note any gaps (what you couldn't find or verify)

## Step 5 — Report Output

Use the template in `ressources/report-template.md`.

Save the finished report to:
`Outputs/[topic-slug]-report-[YYYY-MM-DD].md`

Confirm the file path to the user when done.

## Quality Checks (before delivering)
- [ ] Every claim has a source
- [ ] No unreferenced opinions presented as facts
- [ ] Report matches the audience level from Step 1
- [ ] File saved to Outputs/
